# Legal Clarification Eval Dataset

`legal_clarify_eval.json` is a deterministic synthetic ambiguity/clarification
stress-test set generated by `scripts/build_legal_clarify_eval.py`.

It contains 234 routing-only examples:

- 156 expected `clarify` queries
- 26 expected `dense_retrieval` controls
- 26 expected `graph_traversal` controls
- 26 expected `hybrid_reasoning` controls

Clarification-positive examples cover incomplete context, missing legal entity,
pronoun/reference ambiguity, and multiple-interpretation ambiguity. The dataset
is intended for evaluating whether the two-stage router can trigger
clarification behavior on underspecified legal questions.

Use it as a constructed ambiguity benchmark. Do not describe it as a human
annotated or externally curated dataset unless it is later replaced with such a
dataset.

Regenerate with:

```bash
python scripts/build_legal_clarify_eval.py
```

Evaluate with:

```bash
python scripts/run_clarify_eval.py --config configs/config.yaml --eval-file evaluation/legal_clarify_eval.json --disable-stage2 --output eval_results/clarify_stage1_only.json --csv-output eval_results/clarify_stage1_only.csv
python scripts/run_clarify_eval.py --config configs/config.yaml --eval-file evaluation/legal_clarify_eval.json --output eval_results/clarify_two_stage.json --csv-output eval_results/clarify_two_stage.csv
```
